# DEMO_SCENARIO.md — сквозной сценарий демо (флотационный кейс)

## Цель демо
За 5 минут показать: система из реального отчёта фабрики и литературы ставит диагноз
потерь в тоннах и деньгах, воспроизводит результат экспертного мозгового штурма и находит
новые гипотезы — с кликабельным trace до ячейки исходника и страницы учебника.

## Домен
Флотационное обогащение Cu-Ni руд (данные кейса, `norn-hack/`): потери металлов
(element_28/29 — анонимизированы) с отвальными хвостами 4 обогатительных фабрик.

## KPI демо
```json
{
  "factory_id": "kgmk",
  "target": { "metric": "recoverable_losses_element_28", "direction": "decrease", "minimum_delta_percent": 10 },
  "constraints": [{ "metric": "capex_class", "op": "<=", "value": 3, "unit": "class" }],
  "prices_usd_per_t": { "element_28": 16500, "element_29": 9500 }
}
```
Цены металлов вводит пользователь на демо (анонимизация данных сохраняется).

## Реплики-страховки на вопросы жюри
- «Где здесь LLM?» → только чтение литературы (extract) под pydantic-валидатором;
  диагноз, поиск гипотез, скоринг, деньги — детерминированный Rust. Повторный прогон
  даёт байт-в-байт тот же результат.
- «Почему не предсказательная ML-модель?» → 4 фабрики — это не датасет для обучения;
  граф знаний + детерминированные операторы дают объяснимость, которую требует ТЗ.
  ML-скоринг — в roadmap, когда появится фидбэк-история.
- «Что с неполными данными?» → в КГМК-файле секция +125 битая (#REF!) — система
  показывает это в Data Readiness и продолжает работу (см. панель).
- «Масштабируется на другие переделы?» → pack + factories = данные. Показать
  `packs/flotation-v1.yaml`: вся доменная семантика в одном файле.

## Fallback-план
Бэк упал → фронт работает от `fixtures/board.json` (полный флоу кликабелен).
LLM недоступна → extract идёт из зафиксированного `fixtures/extract_response_full.json`.
Живое демо недоступно → видео записано заранее.

## Демо-данные
- Основной прогон: КГМК (`norn-hack/Пример 1/`), финальный live: ТОФ (`Пример 4/`).

## Скрипт демо через API (backend, без фронта)

Проверенный сквозной прогон по HTTP-шву. Запуск платформы:

```sh
cd backend && NORNIKEL_ROOT=../docs cargo run -p platform      # :8080
```

### 1. Диагноз + портфель + деньги
```sh
curl -sXPOST localhost:8080/run -H 'content-type: application/json' \
     -d '{"factory_id":"kgmk"}'
```
Ждём: `run_id: run_0001`; топ-1 `node_hydrocyclone_nozzle` (насадки ГЦ 12→8),
`value_usd_range ≈ [3 955 050, 11 865 150]`, статус `recommended`; в портфеле есть
gap-гипотезы `node_fine_screening` и `node_magnetic_separation` (оборудования нет на КГМК).
Реплика: *«не "улучшить измельчение", а действия с ценником; крупные ходы с новым
оборудованием система честно помечает как gap»*.

### 2. Benchmark против недельного мозгового штурма
```sh
curl -s localhost:8080/benchmark | jq '{coverage_pct, matched, expert_total}'
```
Ждём: `coverage_pct: 100, matched: 5, expert_total: 5` — **engine переоткрыл все 5
эталонных гипотез экспертов КГМК**. Реплика: *«за 3 минуты воспроизвели неделю
экспертной работы, и каждая гипотеза подтверждена литературой»*.

### 3. Trace до первоисточника
```sh
curl -s localhost:8080/trace/hyp_001 | jq '{claims: [.claims[] | {id, source_page}], top_cell: .source_cells[0]}'
```
Ждём: `claim_001` со `source_page: 212` (учебник флотации) и ячейку `Итог!E44`
(2088 т закрытого Pnt/Cp). Реплика: *«ни одно число не выдумано — клик до страницы и до ячейки xlsx»*.

### 3-бис. Честный roadmap (де-дубликация денег) ⭐
```sh
curl -s localhost:8080/roadmap | jq '{total_value_usd_range, covered_diagnoses, phases: [.phases[] | {label, value_usd_range}]}'
```
Ждём: наивная сумма по 11 гипотезам ≈ **$87M** (двойной счёт — все делят один тоннаж
диагноза), честный `total_value_usd_range ≈ [$4.25M, $12.76M]` по 2 разным механизмам
потерь, и все действия — **capex 1 (быстрые настройки, без капзатрат)**.
`GET /roadmap?max_capex=1` — «бюджетный ползунок». Реплика: *«не складываем все идеи в
$87M — честно: ~$13M, и это ловится настройкой режима, без нового оборудования»*.

### 4. What-if по цене металла (live rerun)
```sh
curl -sXPOST localhost:8080/rerun -H 'content-type: application/json' \
  -d '{"run_id":"run_0001","action":{"kind":"change_price","payload":{"element":"element_28","usd_per_t":33000}}}' \
  | jq '.hypotheses[0].economic_effect.value_usd_range, .snapshot.hash'
```
Ждём: `value_usd_range` удвоился `≈ [7.9M, 23.7M]`, **`snapshot.hash` не изменился**
(граф тот же, пересчёт только денег/ранжирования).

### 5. Универсальность + честность о данных
```sh
curl -s localhost:8080/factories | jq '.[] | {factory_id, opportunity_usd_mid, expert_coverage_pct}'
curl -s localhost:8080/data_readiness | jq '{readiness_pct, issues_by_type}'
```
Ждём: 4 фабрики (kgmk ≈ $16.5M/100%, tof ≈ $17M — 2 секции хвостов, nof_med ≈ $9.2M/87.5%,
nof_vkr ≈ $5.4M/83.3%); Data Readiness КГМК `62.5%` с `ref_error: 33` (битая секция +125
обработана детерминированно). Реплика: *«другая фабрика — тот же движок; про плохие
данные говорим честно»*.

### Финал: воспроизводимость
Повторный `POST /run {"factory_id":"kgmk"}` даёт тот же `snapshot.hash` и байт-в-байт тот же
портфель. `GET /export/board.csv` — плоская выгрузка для жюри.
