# AGENT_RULES.md — жёсткие запреты и требования

Эти правила сильнее любой задачи. Если задача требует их нарушить — задача неверна.

## Запрещено
1. **Доменные слова в ядре.** В `crates/engine` не должно быть ни `alloy`, ни `Sc`,
   ни `strength`, ни любого материаловедения. Только generic-типы (`NodeId`, `EdgeType`,
   `Lever`, `Score`). Ревью каждого PR в engine начинается с вопроса:
   «есть ли тут хоть одно доменное слово?» Есть → выносим в pack.
2. **PyO3 / встраивание Python в Rust.** Граница только HTTP+JSON.
3. **LLM решает.** LLM делает только extract / skeptic / narrate. Генерацию кандидатов,
   scoring, статусы и ранжирование делает Rust-код. Если ранжирование зависит от вывода
   LLM — это баг.
4. **Недетерминизм в ядре.** Одинаковый snapshot+hash обязан давать одинаковые гипотезы.
   Любая случайность — только с фиксированным seed из контракта.
5. **Дубли структур.** Структуры контракта живут только в `crates/contracts`.
   Не переобъявлять их в engine/platform/sidecar/web.
6. **Секреты в репо.** Ключи LLM — только через env, никогда в коммите.

## Требуется
- Rust валидирует ВЕСЬ JSON от Python на входе (serde + явная проверка инвариантов).
- Каждая гипотеза грунтована: нет источника → не топ. Статус ставит код.
- Любое изменение контракта → регенерация TS-типов в одном PR.
- Любое доменное правило/вес → в pack (docs/DOMAIN_PACK.md), не в код.
