# prompt_frontend.md — ночной бриф: Роль 3 (Frontend)

Ты работаешь в проекте «Фабрика гипотез» (хакатон Норникель, кейс: флотация Cu-Ni руд,
снижение потерь металлов с хвостами). Твоя зона: `web/` — React/TypeScript, Vite, порт 5173.
На тебе держится всё демо-видео. Дедлайн сдачи: 4 июля 23:59.

## Контекст кейса за 5 строк
Система ставит фабрике «диагноз потерь» из реального xlsx-отчёта (тонны металла по классам
крупности × минералогии, с причиной), строит гипотезы с деньгами и показывает кликабельный
trace до ячейки исходника и страницы учебника. Коронный номер — benchmark: сравнение
гипотез системы с гипотезами живых экспертов (golden set). Весь UI работает от
`docs/fixtures/board.json` с минуты 0 — бэк не жди.

## P0 задачи (строго по порядку)
1. **Каркас + загрузка**: board из `fixtures/board.json` (import) с переключателем на
   `GET :8080/board` (env-флаг). Строгий TS.
2. **Diagnosis view — героический экран демо.** Heatmap: строки = классы крупности
   (+125 → -10), колонки = минералогические формы, цвет = тонны потерь (element_28
   по умолчанию, переключатель на 29). Данные: `board.diagnostics.loss_cells`.
   Сверху три больших числа: «Потеряно всего N т» (totals), «Извлекаемо M т»
   (сумма recoverable cells), «≈ $X млн/год» (M × price из kpi_contract).
   Ячейки not_recoverable — приглушённые с штриховкой. Панель Data Readiness:
   `diagnostics.data_quality` (иконка + issue + location). Шрифты крупные — экран
   должен читаться с проектора.
3. **Портфель гипотез**: карточки по rank. На карточке: title, status-бейдж,
   `value_usd_range` («$2.5–7.4 млн/год» — форматируй красиво), addressable tons,
   capex-класс (①настройка/②замена/③новое оборудование), score_total.
4. **Карточка гипотезы** (роутинг или модалка): summary → механизм (source_nodes цепочкой) →
   **trace**: клик по claim-id → цитата claim.text + документ + `source_page`; клик по
   diag-ссылке → фрагмент heatmap с подсветкой соответствующей ячейки (`cell_ref` покажи
   текстом «Итог!E44 исходного отчёта»). Score breakdown (6 осей, bar-чарт).
   `economic_effect` с раскрытыми assumptions. Риски, missing_evidence, DOE-план.
5. **Benchmark view — киллер-фича.** Два столбца: слева гипотезы экспертов
   (`golden/expert_hypotheses.json`, фильтр по factory_id), справа — гипотезы системы.
   Совпавшие (по `expert_match`, а до его появления — вручную сматченные пары по
   lever_type) соединены визуально и подсвечены зелёным; новые гипотезы системы — синий
   бейдж «нет у экспертов»; непокрытые экспертные — серые. Сверху крупно:
   «Воспроизведено M из N гипотез мозгового штурма + K новых».
6. **Rerun-контролы** (кнопки над портфелем): исключить фактор (× на карточке),
   слайдер веса cost, **поле «цена металла, $/т»** → `POST /rerun` change_price
   (от фикстуры — локальный пересчёт value_usd_range тем же правилом:
   tons × gain/100 × price). Подсветка изменившихся позиций после rerun.
7. **Переключатель фабрики** (kgmk / nof_vkr / nof_med / tof) в шапке — для финала демо.
   От фикстур: диагностика есть для всех 4 (`diagnostics_*.json`), board — для kgmk;
   для остальных показывай диагноз + заглушку «run pending» до готовности бэка.
8. **Кнопка «Экспорт»** → `GET :8080/export/board.csv` (или локальная генерация CSV
   из board — 20 строк кода, требование ТЗ).

## Правила
- Ничего не придумывать: только данные из BoardResponse. Нет поля — нет виджета.
- Все суммы с разрядами и валютой, тонны с округлением до целых.
- Работа без бэка обязательна до самого конца (демо-страховка).

## Самопроверка (Done)
- Полный флоу кликается от fixtures без запущенного бэка: диагноз → портфель →
  карточка → trace → benchmark → rerun → экспорт.
- Карточка гипотезы отвечает на 4 вопроса: почему? (механизм+trace) откуда? (источник+страница)
  сколько денег? (economic_effect) как проверить? (DOE).
- Heatmap читается с 3 метров: контраст, подписи классов и форм по-русски.
- `tsc --noEmit` зелёный, никаких `any` в типах контракта.
