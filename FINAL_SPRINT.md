## Что изменилось и почему пересборка

Организаторы выложили реальные данные кейса (`norn-hack/`). Это **не сплавы** — это **флотация медно-никелевых руд**, потери металлов с хвостами обогатительных фабрик:

- **4 кейса** (КГМК, НОФ-вкр, НОФ-мед, ТОФ): пара «xlsx с анализом хвостов» + «docx с гипотезами экспертов»
- «Элемент 28» = Ni, «Элемент 29» = Cu (анонимизировано — в коде НЕ раскрывать, работаем с `element_28/29`, цена — параметр)
- 5 PDF-учебников по флотации/обогащению — корпус для RAG
- PNG-схемы флотации и списки оборудования фабрик
- `Как читать отчет института по хвостам.docx` — **готовый алгоритм диагностики**, наш детерминированный движок в чистом виде

**Главная находка:** docx «Гипотезы» — это ground truth. Мы можем измеримо показать: «система воспроизвела N из M гипотез экспертного мозгового штурма и нашла новые». Ни одна команда этого не сделает.

**Вторая находка:** организаторы в чате подтвердили — можно и нужно использовать открытые источники данных, «нам очень интересны решения, которые расширены таким образом». Наш RAG-корпус расширяется открытыми статьями, novelty гипотез считается против открытого корпуса. Это бонусные очки, встроено в роли ниже.

**Архитектура НЕ меняется.** Claims → граф → snapshot → детерминированные операторы → scoring → trace. Меняются: domain pack (`flotation-v1`), вход (+ детерминированный парсер xlsx) и экономический слой в скоринге.

## Решение одной схемой

```
xlsx хвостов ──► /diagnose (Python, БЕЗ LLM) ──► LossMatrix: [класс крупности × минералогия]
                                                  → тонны потерь, извлекаемость, диагноз
PDF-корпус  ──► /extract (Python, LLM+RAG) ──► claims: рычаг → механизм → диагноз → KPI
(5 учебников + открытые статьи)                   (с источником: документ, страница)

LossMatrix + claims + flotation-v1.yaml + factories/*.yaml
        ──► Rust engine: mechanism_path от KPI назад через диагнозы с ненулевыми тоннами
             до controllable-рычагов, доступных на данной фабрике
        ──► Hypothesis { trace, score, economic_effect ($ и тонны), doe_plan }
        ──► Frontend: heatmap диагноза · портфель с $ · карточка с trace · benchmark vs эксперты
```

Логика диагностики (из «Как читать отчет», кодируется в pack, не в engine):

| Паттерн в LossMatrix | Диагноз (id) | Классы решений из эталонных гипотез |
|---|---|---|
| Закрытый Pnt/Cp в крупных классах (+125, +71, −71+45) | `liberation_deficit` | доизмельчение, футеровка, шары, насадки гидроциклонов, грохота |
| Раскрытый Pnt/Cp в классе −10 | `slimes_overgrinding` | контроль гранулометрии, классификация, не переизмельчать |
| Раскрытый Pnt/Cp в средних классах | `flotation_kinetics` | время агитации, плотность пульпы, реагенты, фронт флотации |
| Силикатная форма / примесь в пирротине / (пирит для эл.28) | `not_recoverable` | честно отсекается из потенциала |

Извлекаемость (из методички): эл.28 — раскрытый Pnt, закрытый Pnt, миллерит; эл.29 — только раскрытый/закрытый Pnt/Cp. Пороги «крупный/средний/тонкий» — в pack.

## Формула шока для жюри (держим в голове каждую минуту)

1. **Диагноз с деньгами.** «Из 10 392 т Ni в хвостах КГМК извлекаемо ~X тыс. т ≈ $Y млн/год — вот heatmap, где они сидят». Детерминизм, ноль LLM.
2. **Гипотезы с тоннажом.** Не «улучшить измельчение», а «насадки гидроциклонов 12→8 адресуют закрытый Pnt/Cp в +71/−71+45 = 3 005 т Ni; рост раскрытия 5–15% (учебник, стр. N) → $2.4–7.2 млн/год».
3. **Trace до ячейки.** Клик по гипотезе → конкретная ячейка исходного xlsx + страница PDF с механизмом.
4. **Benchmark против экспертов.** Side-by-side: гипотезы системы vs мозговой штурм КГМК. Совпадения, новые, спорные. Затем live-прогон на ТОФ, «который система не видела», за 30 секунд.
5. **Открытые источники.** «Граф знаний собран из учебников кейса + N открытых работ; novelty гипотезы = непокрытость в открытом корпусе» — прямой ответ на пожелание организаторов.

## Роль 1 — Rust ядро (contracts + engine + platform)

**Миссия:** детерминированный движок, который из LossMatrix + claims + pack собирает гипотезы с деньгами и trace. Engine по-прежнему не знает слов «флотация» и «никель».

**P0 (по порядку):**
1. Внести дельты в `crates/contracts` (DiagnosticsReport, economic_effect, prices, factory_id) + регенерить `web/src/contracts.ts` (ts-rs). Это блокер для всех — сделать первым.
2. `GET /board` из `fixtures/board.json` — фронт отвязан.
3. Построение графа: claims → petgraph; диагностические узлы получают `addressable_tons` из DiagnosticsReport; рычаги помечаются `available` по factories/*.yaml.
4. `mechanism_path`: от KPI назад через diagnosis-узлы с tons > 0 до controllable+available рычагов. Гипотеза наследует addressable_tons своего диагноза.
5. Scoring: `kpi_impact ∝ addressable_tons × mid(recovery_gain_range) × price` (нормировать на общие потери); остальные оси по SCORING.md. Хард-фильтры: equipment_not_available, capex_class > лимита из контракта.
6. `economic_effect` — вычисляется в engine (диапазон, не точка; assumptions заполнять).
7. `POST /run`, `POST /rerun` (exclude_factor, change_weight, смена цены металла), `GET /hypothesis/:id`.
8. `GET /export/board.json` + `GET /export/board.csv` — 30 минут, закрывает обязательное требование ТЗ «экспорт CSV/JSON».
9. `gap`-оператор: диагноз с tons > порога, к которому не ведёт ни один доступный рычаг → гипотеза «нужен новый метод/оборудование» (у экспертов КГМК это «магнитная сепарация» — шанс попасть в неё).

**Stretch:** skeptic-rule «capex-доминирование» (гипотеза A даёт ≥80% эффекта гипотезы B при capex_class ниже → флаг на B).

**DoD:** фиксированные diagnostics+claims+pack дают стабильный board с одинаковым hash; в top-5 по КГМК есть гипотезы про доизмельчение/классификацию (сверить с golden); rerun с ценой металла ×2 меняет ranking; в engine нет доменных слов (grep-проверка: `flotation|nickel|pnt|мельниц` в crates/engine — пусто).

**Не делать:** Tauri, contradiction/analogy-код, sqlite-миграции (in-memory + файлы достаточно), NLP-чат.
